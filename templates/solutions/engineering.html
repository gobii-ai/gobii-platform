create_agent.py 
import requests

API_URL = "https://gobii.ai/api/v1/agents/"

# Deploy an always-on AI employee
response = requests.post(
    API_URL,
    headers={"X-Api-Key": API_KEY},
    json={
        "name": "Market Monitor",
        "charter": "Track competitor launches and brief the product team.",
        "schedule": "@daily",
    }
)

response.raise_for_status()
agent = response.json()
print(agent["id"], agent["life_state"])

What is an AI Agent API?

An AI Agent API lets a product deploy and manage AI employees programmatically. Gobii's Agent API is built around persistent agents: create an employee once, assign its charter and schedule, connect tools, send it messages, inspect its timeline and processing state, update its configuration, or pause it without deleting its work.

How do I deploy an always-on AI employee?

Send a POST request to /api/v1/agents/ with a name, charter, and optional cron-like schedule. Gobii returns the employee's ID and lifecycle state. You can then message it, read its timeline, check processing status, change its schedule or tools, and activate or deactivate it through that stable resource.

What can I manage through the Agent API?

The API covers creation, retrieval, updates, soft deletion, activation, deactivation, schedule validation, inbound messages, timelines, processing status, and recent browser tasks. Agent configuration can include a charter, schedule, contact policy, preferred intelligence tier, daily credit limit, templates, and enabled MCP servers. Organization API keys can manage organization-owned agents separately from personal agents.

How do schedules, messages, and webhooks keep employees always on?

A schedule wakes an employee for recurring responsibilities, while inbound messages and webhooks let external systems trigger work when something changes. Gobii queues that work durably and preserves the employee's configuration and workspace between interactions. Your product can fetch the timeline or processing snapshot instead of holding a synchronous request open.

Can Gobii employees use browsers and other tools?

Yes. Browser automation is available for work across rendered websites and authenticated sessions, but it is not the whole platform. Employees can also use connected app tools, MCP servers, files, an embedded SQLite workspace, email, SMS, and webhooks. Their work can produce timeline events, structured data, files, screenshots, database records, and downstream webhook payloads.

When should teams use Gobii Cloud API versus self-hosting?

Use Gobii Cloud when you want the fastest path to production without operating the agent runtime, scheduler, browsers, storage, and upgrades. Self-host the open-source platform when you need deeper infrastructure control, private network access, data residency, or customization. The API model stays centered on the same persistent agent resource in either environment.

What data and privacy controls matter for API agent workflows?

Scope each employee to the tools, credentials, contacts, and data it needs. Use organization ownership and agent-level tool configuration to separate access, set credit limits to bound usage, and inspect timelines or deliverables for important work. Keep human approval around customer-facing, financial, destructive, or otherwise sensitive actions.
