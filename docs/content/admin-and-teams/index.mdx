Admin and team docs are for workspace owners, organization admins, and operators who manage more than one Gobii or more than one user.

## What admins manage

- Organization membership, seats, roles, and billing context.
- Which Gobiis belong to a person or organization.
- Collaborator access and ownership transfers.
- Contact access, allowlists, and invites.
- Global or organization secrets.
- Organization-scoped MCP servers and connected capabilities.
- System skills and profiles where available.
- Usage, billing, task packs, contact packs, dedicated IPs, and add-ons.

## Recommended admin setup

1. Create or select the organization.
2. Add seats and invite members.
3. Decide which Gobiis belong to the organization.
4. Configure shared secrets and MCP servers only where needed.
5. Set contact and collaborator policies.
6. Review usage and billing after the first real workflows run.

For ordinary day-to-day work inside a single Gobii, see [Using Gobii](/using-gobii).
