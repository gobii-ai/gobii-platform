Every account has a pool of _Task Credits_ per billing cycle. Agents consume these credits as they perform tasks, at a granular level. For example, an agent may use 0.04 task credits to send an email. These are deducted from the users task credit allowance upon execution.

The number of monthly credits you receive vary by plan, or enterprise contract. Additionally, paid accounts have the option to enable additional tasks past their monthly allowance, on a per-task credit basis. At the time of this writing, additional tasks are 10¢ each. The additional usage is billed at the end of the month.

## Agent Budgets

Each agent has a budget for the number of task credits it may use in a day. In the user interface, you will see it labeled as a "soft limit." It is labeled soft as Gobii will do its best to complete its assigned function and may go slightly over the limit if your agent is near completion of the current cycle. For example, an agent with a soft limit of 5 may end up using 5.2 task credits in a day. The user may set the soft limit on an agent-by-agent basis.

Internally, Gobii will pause functioning until the next day when the _hard limit_ is reached. The hard limit is a function of the soft limit, and is not directly configurable.

## Usage

Usage may be monitored on the dedicated [usage](https://gobii.ai/console/usage/) section of your account or organization.
