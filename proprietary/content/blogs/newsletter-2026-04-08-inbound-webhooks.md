Most people do not need more AI tools. They need the ones they already use to actually talk to each other.

Inbound webhooks make that possible in Gobii. Instead of manually prompting an agent every time something happens, you can trigger an agent automatically from events across your existing software stack.

When another tool sends a webhook, a Gobii agent can pick up the context and start working right away.

## Why Reactive Agents Matter

Most business workflows start with an event: a form submission, a completed meeting recording, a new CRM record, a support ticket, a signed contract, or a spreadsheet update.

Without webhooks, someone has to notice the event and tell an agent what to do next. That handoff is slow, easy to forget, and hard to scale.

With inbound webhooks, the handoff becomes automatic. Your existing tools can wake up a Gobii agent the moment work is ready.

## How It Works

1. An external system sends a webhook to Gobii.
2. Gobii receives the payload and routes it to the right agent.
3. The agent uses the webhook data as context for its task.
4. The agent can research, write, update records, notify teammates, or trigger the next step.

That turns agents from tools you manually check into workers that respond to live events.

## Real-World Example: The Meeting Handoff

Imagine your AI meeting notetaker finishes processing a client call. It fires a webhook straight to your Gobii agent.

Your agent instantly:

- drafts a personalized follow-up email
- updates the deal stage in Salesforce
- schedules a reminder task

The meeting ends, the webhook fires, and the follow-up work starts without a manual prompt.

## Other Useful Triggers

Inbound webhooks are useful anywhere another system already knows something happened.

- A lead form submission can trigger research and a personalized first-touch email.
- A support ticket can trigger account research and a suggested response.
- A billing event can trigger internal follow-up or customer communication.
- A spreadsheet update can trigger enrichment, validation, or reporting.
- A project-management event can trigger a status summary or handoff message.

If a tool can send a webhook, it can now hand work to a Gobii agent.
